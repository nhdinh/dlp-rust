## Why The Fix May Be Insufficient

`IOCTL_DISK_SET_DISK_ATTRIBUTES` with `DISK_ATTRIBUTE_READ_ONLY` is partially advisory in some Windows versions. Microsoft docs note the flag may not block all write paths. Likely scenarios:
- The IOCTL succeeds at the partition manager level but the file system above (NTFS/FAT) doesn't honor it
- The IOCTL fails silently due to permissions (needs SYSTEM/admin to open `\\.\PhysicalDriveN` with `GENERIC_READ_WRITE`)
- The 500ms settle window in `enforce_read_only_on_arrival` is too short — drive mounted with write cache before flag applied
