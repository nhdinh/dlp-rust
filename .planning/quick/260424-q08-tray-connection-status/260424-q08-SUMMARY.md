# Phase 260424-q08 Plan 01: Tray Connection Status Summary

Live connection-state tray tooltip showing UI->Agent and Agent->Server states, updated every 100ms via existing Tick mechanism.

| 3 | Tooltip rebuild on every Tick | 16c045e | app.rs |
| 4 | Broadcast ServerConnected after heartbeat | a414413 | offline.rs, messages.rs (fmt) |

## What Was Built

**Task 1** — `Pipe2AgentMsg::ServerConnected { connected: bool }` added to both `dlp-user-ui/src/ipc/messages.rs` and `dlp-agent/src/ipc/messages.rs`. The new variant is handled immediately in `dlp-user-ui/src/ipc/pipe2.rs` by storing the value in `AGENT_SERVER_CONNECTED: AtomicBool`. Serde roundtrip unit tests added and passing in both crates.

**Task 2** — `dlp-user-ui/src/tray.rs` gains:
- `PENDING_TOOLTIP: parking_lot::Mutex<Option<String>>` — cross-thread tooltip queue
- `TRAY_ICON: thread_local! RefCell<Option<TrayIcon>>` — replaces `std::mem::forget(tray)`
- `update_tooltip(text: &str)` — queues tooltip from any thread
- `apply_pending_tooltip()` — drains queue and calls `set_tooltip` on main thread only

**Task 3** — `dlp-user-ui/src/app.rs` Tick arm now rebuilds the tooltip every 100ms using `pipe1_connected` (RwLock) and `ipc::pipe2::agent_server_connected()` (AtomicBool), then calls `update_tooltip` + `apply_pending_tooltip`. Format: `"DLP Agent UI\nUI->Agent: {state}\nAgent->Server: {state}"`.

**Task 4** — `dlp-agent/src/offline.rs` `heartbeat_loop` now captures `send_heartbeat().await.is_ok()` and broadcasts `Pipe2AgentMsg::ServerConnected { connected }` via `BROADCASTER` after every heartbeat attempt (success or failure).

## Deviations from Plan

**1. [Rule 1 - Bug] ServerConnected handler added in Task 1, not Task 2**
- **Found during:** Task 1 — adding the variant to `Pipe2AgentMsg` immediately caused a non-exhaustive match in `pipe2.rs`
- **Fix:** Added `AGENT_SERVER_CONNECTED` atomic and the `ServerConnected` match arm in `pipe2.rs` as part of Task 1 commit (avoiding a non-compiling intermediate state)
- **Files modified:** `dlp-user-ui/src/ipc/pipe2.rs`
- **Commit:** a7d31e4

**2. [Rule 1 - Formatting] cargo fmt applied after Task 4**
- **Found during:** Post-Task 4 `cargo fmt --check` — two files had minor formatting differences (line-wrap in assert_matches, static declaration)
- **Fix:** Ran `cargo fmt` across workspace; no logic changes
- **Files modified:** `dlp-agent/src/ipc/messages.rs`, `dlp-agent/src/service.rs`
- **Commit:** a414413 (bundled with Task 4)

**3. Heartbeat broadcast placed in `offline.rs`, not `service.rs`**
- **Reason:** `send_heartbeat()` is called inside `OfflineManager::heartbeat_loop` in `offline.rs`. The plan mentioned `service.rs` as the target file but the actual heartbeat call site is in `offline.rs`. Broadcasting there avoids duplicating logic.
- **Impact:** None — same observable behavior. BROADCASTER is the same static.

## Known Stubs

None. All connection states are wired to live data sources:

- `agent_server_connected()` — `AtomicBool` updated by real IPC messages from the agent
