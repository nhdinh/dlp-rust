<objective>
Add a self-termination watchdog to dlp-user-ui that exits the process when the
pipe1 connection drops after having been established at least once.

Purpose: If dlp-agent is killed via `taskkill`, the UI has no agent to talk to
and should not linger. The "ever connected" latch prevents a false-positive exit
during the startup window before the agent attaches.

Output: Modified `dlp-user-ui/src/app.rs` with `pipe1_ever_connected` field and
self-kill logic in the `Tick` arm.
</objective>

Make three targeted changes to `dlp-user-ui/src/app.rs`:

**1. Add field to `UiState` (after `pipe1_connected`):**

```rust
struct UiState {
    session_id: u32,
    pipe1_connected: Arc<RwLock<bool>>,
    /// Latches to `true` the first time `pipe1_connected` is seen as `true`.
    /// Used to distinguish "agent not yet started" (safe to wait) from
    /// "agent was running and died" (self-terminate).
    pipe1_ever_connected: bool,
}
```

**2. Initialise the field in `run_with` (per `UiState` struct literal):**

```rust
let state = UiState {
    session_id,
    pipe1_connected,
    pipe1_ever_connected: false,
};
```

**3. Add watchdog block in `update()` inside the `Message::Tick` arm, AFTER the
tray poll block and BEFORE the final `iced::Task::none()` return:**

```rust
// Watchdog: if pipe1 was ever connected and is now disconnected,
// the agent has been killed — terminate the UI process.
let currently_connected = *self.state.pipe1_connected.read();
if currently_connected {
    self.state.pipe1_ever_connected = true;
} else if self.state.pipe1_ever_connected {
    info!("dlp-agent disconnected — self-terminating");
    std::process::exit(0);
}
```

Do NOT modify `spawn_ipc_tasks` or any other function — the latch is maintained
entirely from the `Tick` arm, which polls at 100 ms. The `pipe1_connected` flag
is already set to `false` by the background task when `connect_and_run` returns
(line 70 in current source).
