# Phase 38.2 Plan 02: WR-01 Race + Startup Gap Fix Summary

**USB identity reconciliation heuristic fixes VOLUME-before-USB_DEVICE race and startup enforcement gap so Blocked-tier USB devices are enforced on ALL plug-in timing paths**

## Accomplishments

- WR-01 race fix: `on_usb_device_arrival` now checks for exactly one unmapped drive in `blocked_drives` and reconciles the identity immediately, instead of always parking and missing the case where VOLUME already arrived
- Startup gap fix: `scan_existing_usb_identities` enumerates present USB mass-storage devices at startup and applies tier enforcement for each, covering plug-before-agent and agent-restart-while-plugged paths
- Shared `reconcile_identity_with_unmapped_drive` helper eliminates code duplication between arrival and startup paths
- 6 new unit tests cover reconciliation, parking, ambiguous cases, and already-mapped skips

1. **Task 1: Fix WR-01 race in on_usb_device_arrival** - `f0b171d` (feat)

## Files Created/Modified

- `dlp-agent/src/detection/usb.rs` - WR-01 fix in `on_usb_device_arrival`, new `scan_existing_usb_identities` method, `reconcile_identity_with_unmapped_drive` helper, 6 new unit tests
- `dlp-agent/src/service.rs` - Added `detector.scan_existing_usb_identities()` call after `scan_existing_drives()` at startup

## Decisions Made

- Heuristic only fires for exactly one unmapped drive: zero means VOLUME has not arrived yet (park normally); multiple means ambiguous (park rather than guess wrong). This is the safe default per T-38.2-02-01 threat mitigation.
- `scan_existing_usb_identities` reuses the existing `dlp_common::usb::enumerate_connected_usb_devices` function instead of adding new Win32 SetupDi calls, keeping the change minimal and testable.
- `pending_identity` is explicitly cleared after reconciliation to prevent double-processing when a subsequent VOLUME event arrives.
- Already-mapped identity check (VID/PID/serial triple match) prevents overwriting existing entries during startup scan, handling the race where an arrival notification fires concurrently with the startup enumeration.
