        {
            warn!(
                vid = %identity.vid,

                "USB device disabled (Blocked tier)"

        // Layer 2: DACL deny-all (defense-in-depth fallback).
        if let Err(e) = controller.set_volume_deny_all(letter) {

|-----------|----------|-----------|-------------|-----------------|
| T-38.2-01-01 | Denial of Service | set_volume_deny_all SDDL | mitigate | SDDL explicitly allows SYSTEM (S-1-5-18) full access; OS operations on volume are preserved. Denies only Everyone and Authenticated Users. |
| T-38.2-01-02 | Elevation of Privilege | DACL applied to wrong volume | mitigate | Drive letter comes from VOLUME notification (kernel-assigned) or from handle_volume_event rescan; not user-controlled. Same source as existing set_volume_readonly. |
| T-38.2-01-03 | Tampering | Original DACL not restored | mitigate | Both set_volume_readonly and set_volume_deny_all use the same original_dacls cache and the same restore_volume_acl cleanup path. Removal handler always calls restore_volume_acl. |
| T-38.2-01-04 | Denial of Service | Admin locked out of volume | accept | Admin can stop the dlp-agent service to clear the DACL (restore_volume_acl is called on removal, but if service crashes, restart calls scan_existing_drives which may re-apply).
