Create `scripts/Uat-UsbBlock.ps1` with the following structure:

1. **Requires statement**: `#Requires -RunAsAdministrator` (IOCTL operations require elevation).

2. **Param block**:
   ```powershell
   param(
       [string]$ServerUrl = "http://127.0.0.1:9090",
       [string]$JwtToken = $env:DLP_ADMIN_JWT,
       [switch]$SkipBlockedTest,
       [switch]$SkipReadOnlyTest
   )
   ```

3. **Helper: `Get-RemovableUsbDrives()`**:
   - Query `Get-WmiObject -Class Win32_DiskDrive | Where-Object { $_.MediaType -eq "Removable media" -or $_.InterfaceType -eq "USB" }`.
   - For each drive, also query `Win32_DiskDriveToDiskPartition` and `Win32_LogicalDiskToPartition` to get the drive letter.
   - Return array of objects with: Index, DeviceID, Model, Size, DriveLetter, SerialNumber (from `Win32_DiskDrive.SerialNumber` trimmed).

4. **Helper: `Show-DriveMenu($drives)`**:
   - Print numbered menu: "1: Drive E: - SanDisk Cruzer (14.9 GB)".
   - Prompt user to enter a number.
   - Validate input and return selected drive object.

5. **Helper: `Register-Device($drive, $tier)`**:
   - Build POST body with vid, pid, serial, description, trust_tier.
   - Use `Invoke-RestMethod -Uri "$ServerUrl/admin/device-registry" -Method POST -Headers @{ Authorization = "Bearer $JwtToken" } -ContentType "application/json" -Body $body`.
   - Return the response (contains the device id).

6. **Helper: `Remove-Device($id)`**:
   - DELETE `$ServerUrl/admin/device-registry/$id` with JWT header.

7. **Helper: `Test-WriteBlocked($driveLetter)`**:
   - Create a temporary file path on the drive: `$driveLetter:\UatTestWrite.tmp`.
   - Try to write a small test string using `[System.IO.File]::WriteAllText($testPath, "test")`.
   - Catch the exception. If the HResult is 0x80070013 (ERROR_WRITE_PROTECT) or the message contains "write-protect", return `$true`.
   - Otherwise, return `$false`.
   - Clean up the test file if it was somehow created.

8. **Helper: `Test-ReadAllowed($driveLetter)`**:
   - Create a test file first (before setting read_only) using direct Win32 API or by temporarily setting full_access.
   - ALTERNATIVE: Use a file that already exists on the drive (e.g., a small text file the user creates beforehand).
   - SIMPLER: The script will:
     a. First test with `blocked` tier — no read/write allowed.
     b. Then change tier to `read_only` via PUT (upsert) — reads allowed, writes denied.
     c. Verify read by trying to list directory contents: `Get-ChildItem "$driveLetter:\"`.
     d. Verify write is blocked using the same Test-WriteBlocked helper.

9. **Helper: `Clear-DiskReadOnly($driveIndex)`**:
   - Use `DeviceIoControl` with `IOCTL_DISK_SET_DISK_ATTRIBUTES` to clear the read-only attribute.
   - This requires P/Invoke in PowerShell. Use `Add-Type` with C# code that calls `CreateFileW` and `DeviceIoControl`.
   - The C# code should:
     - Open the physical drive with `\\.\PhysicalDriveN`.
     - Call `DeviceIoControl` with control code `0x7C104` (IOCTL_DISK_SET_DISK_ATTRIBUTES).
     - Pass a struct with `AttributesMask = DISK_ATTRIBUTE_READ_ONLY (0x00000001)` and `Attributes = 0`.
   - Return `$true` on success, `$false` on failure.

10. **Main script flow**:
    ```powershell
    Write-Host "=== DLP USB Write-Protection UAT ===" -ForegroundColor Cyan

    # Validate JWT
    if (-not $JwtToken) { Write-Error "DLP_ADMIN_JWT env var or -JwtToken param required"; exit 1 }

    # Detect drives
    $drives = Get-RemovableUsbDrives
    if ($drives.Count -eq 0) { Write-Error "No removable USB drives detected"; exit 1 }

    # Select drive
    $selected = Show-DriveMenu $drives
    $driveLetter = $selected.DriveLetter
    $driveIndex = $selected.Index

    # Extract VID/PID from PNPDeviceID if available, or use placeholders
    $vid = "0000"; $pid = "0000"; $serial = $selected.SerialNumber.Trim()
    if ($selected.PNPDeviceID -match "VID_([0-9A-F]{4})&PID_([0-9A-F]{4})") {
        $vid = $Matches[1]; $pid = $Matches[2]
    }

    $passCount = 0; $failCount = 0

    # --- Blocked tier test ---
    if (-not $SkipBlockedTest) {
        Write-Host "`n[Test] Blocked tier..." -ForegroundColor Yellow
        $device = Register-Device $selected "blocked"
        Start-Sleep -Seconds 2  # Allow agent to poll

        $writeBlocked = Test-WriteBlocked $driveLetter
        if ($writeBlocked) {
            Write-Host "  PASS: Write blocked (ERROR_WRITE_PROTECT)" -ForegroundColor Green
            $passCount++
        } else {
            Write-Host "  FAIL: Write was NOT blocked" -ForegroundColor Red
            $failCount++
        }

        Remove-Device $device.id
    }

    # --- ReadOnly tier test ---
    if (-not $SkipReadOnlyTest) {
        Write-Host "`n[Test] ReadOnly tier..." -ForegroundColor Yellow
        $device = Register-Device $selected "read_only"
        Start-Sleep -Seconds 2

        $readAllowed = $false
        try {
            $items = Get-ChildItem "$driveLetter:\" -ErrorAction Stop
            $readAllowed = $true
            Write-Host "  PASS: Read allowed ($($items.Count) items listed)" -ForegroundColor Green
            $passCount++
        } catch {
            Write-Host "  FAIL: Read was blocked: $_" -ForegroundColor Red
            $failCount++
        }

        $writeBlocked = Test-WriteBlocked $driveLetter
        if ($writeBlocked) {
            Write-Host "  PASS: Write blocked on read-only device" -ForegroundColor Green
            $passCount++
        } else {
            Write-Host "  FAIL: Write was NOT blocked on read-only device" -ForegroundColor Red
            $failCount++
        }

        Remove-Device $device.id
    }

    # --- Cleanup ---
    Write-Host "`n[Cleanup] Clearing disk read-only attributes..." -ForegroundColor Yellow
    $cleared = Clear-DiskReadOnly $driveIndex
    if ($cleared) {
        Write-Host "  PASS: Disk attributes cleared" -ForegroundColor Green
    } else {
        Write-Host "  WARN: Failed to clear disk attributes (may need manual cleanup)" -ForegroundColor Yellow
    }

    # --- Summary ---
    Write-Host "`n=== Results ===" -ForegroundColor Cyan
    Write-Host "PASS: $passCount" -ForegroundColor Green
    Write-Host "FAIL: $failCount" -ForegroundColor Red
    if ($failCount -gt 0) { exit 1 } else { exit 0 }
    ```

11. **Color coding**:
    - PASS: Green (`Write-Host ... -ForegroundColor Green`)
    - FAIL: Red (`Write-Host ... -ForegroundColor Red`)
    - INFO: Cyan/Yellow (`Write-Host ... -ForegroundColor Cyan`)

12. **Error handling**: Wrap each API call and disk operation in `try/catch`. On error, print the exception message and continue to cleanup.
