2. **Helper: `spawn_server(db_path: &Path, jwt_secret: &str) -> (std::process::Child, u16)`**:
   - Build Command: `cargo run --bin dlp-server --`.
   - Set env vars:
     - `DLP_DB_PATH` = db_path.to_str().unwrap()
     - `DLP_JWT_SECRET` = jwt_secret
     - `DLP_BIND_ADDR` = "127.0.0.1:0" (let OS assign port)
   - Set `CARGO_TARGET_DIR` = "target-test".
   - Redirect stdout/stderr to Stdio::null().
   - Spawn the process.
   - Poll stdout for "Listening on" line to extract port (or use a fixed port and retry). ALTERNATIVE: Use a fixed high port (e.g., 19090) and retry if in use.
   - Return (child, port).

   SIMPLER APPROACH (per D-10 in RESEARCH.md): Use `TcpListener::bind("127.0.0.1:0")` in the test to get a free port, then pass that port to the server via env var. But the server may not accept a pre-assigned port. INSTEAD: Parse the server's startup log for the actual port.

   PRACTICAL APPROACH: Use the in-process router (same as dlp-server tests) instead of spawning the binary for the server side. The server runs in-process; only the agent is spawned as a binary. This avoids port discovery issues and is faster.
   - Call `dlp_e2e::helpers::server::build_test_app()` to get router + pool.
   - Bind TCP listener to 127.0.0.1:0.
   - Spawn `axum::serve(listener, router)` in a background tokio task.
   - Get the local port from `listener.local_addr()`.
   - This gives us the server port without parsing logs.

3. **Helper: `spawn_agent(config_path: &Path, server_url: &str) -> std::process::Child`**:
   - Build Command: `cargo run --bin dlp-agent --`.
   - Set env vars:

     - `DLP_SERVER_URL` = server_url
   - Set `CARGO_TARGET_DIR` = "target-test".
   - Redirect stdout/stderr to Stdio::null().
   - Spawn and return the Child.

4. **Helper: `seed_global_agent_config(port: u16, payload: serde_json::Value)`**:
   - Create reqwest::blocking::Client.
   - PUT to `http://127.0.0.1:{port}/admin/agent-config`.
   - Header: `Authorization: Bearer {minted_jwt}`.
   - Body: JSON payload.
   - Assert status is 200.

5. **Helper: `poll_config_file(config_path: &Path, timeout: Duration) -> Option<AgentConfig>`**:
   - Loop every 500ms until timeout.
   - If file exists, try AgentConfig::load(config_path).
   - Return Some(config) on successful load.
   - Return None if timeout expires.
