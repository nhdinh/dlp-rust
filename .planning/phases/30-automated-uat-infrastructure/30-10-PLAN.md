Create `.github/workflows/nightly.yml`:

1. **Workflow trigger**: `on: schedule: - cron: '0 2 * * *'` (runs at 2 AM UTC daily). Also add `workflow_dispatch:` for manual trigger.

2. **Job: `release-build-and-smoke`**:
   ```yaml
   name: Nightly Release Build + Smoke Tests

   on:
     schedule:
       - cron: '0 2 * * *'
     workflow_dispatch:

   jobs:
     release-build-and-smoke:
       name: Release Build and Smoke Tests
       runs-on: windows-latest
       steps:
         - uses: actions/checkout@v4

         - name: Install Rust toolchain
           uses: dtolnay/rust-action@stable

         - name: Cache cargo registry
           uses: actions/cache@v4
           with:
             path: |
               ~/.cargo/registry
               ~/.cargo/git
               target-release
             key: ${{ runner.os }}-cargo-release-${{ hashFiles('**/Cargo.lock') }}

         - name: Build workspace in release mode
           run: cargo build --workspace --release
           env:
             CARGO_TARGET_DIR: target-release
             RUSTFLAGS: "-D warnings"

         - name: Run clippy on release build
           run: cargo clippy --workspace --release -- -D warnings
           env:
             CARGO_TARGET_DIR: target-release

         - name: Run workspace tests against release binaries
           run: cargo test --workspace --release
           env:
             CARGO_TARGET_DIR: target-release
             RUSTFLAGS: "-D warnings"

         - name: Smoke test — verify release binaries exist
           run: |
             Test-Path target-release/release/dlp-server.exe
             Test-Path target-release/release/dlp-agent.exe
             Test-Path target-release/release/dlp-admin-cli.exe
           shell: pwsh

         - name: Smoke test — dlp-server health check
           run: |
             $proc = Start-Process -FilePath "target-release/release/dlp-server.exe" -PassThru -WindowStyle Hidden
             Start-Sleep -Seconds 5
             try {
               $resp = Invoke-RestMethod -Uri "http://127.0.0.1:9090/health" -TimeoutSec 5
               if ($resp.status -eq "ok") {
                 Write-Host "PASS: dlp-server health check" -ForegroundColor Green
               } else {
                 Write-Error "FAIL: dlp-server health check returned unexpected status"
               }
             } catch {
               Write-Error "FAIL: dlp-server health check failed: $_"
             } finally {
               Stop-Process -Id $proc.Id -Force -ErrorAction SilentlyContinue
             }
           shell: pwsh
   ```

3. **Key requirements**:
   - `CARGO_TARGET_DIR: target-release` for all cargo commands (D-23)
   - `RUSTFLAGS: "-D warnings"` for build and test (D-21)
   - `runs-on: windows-latest` (D-13)
   - Scheduled trigger only (D-22 — not on PR/push)
   - Smoke test verifies release binaries exist and dlp-server responds to health check
   - Cache uses `target-release` path

4. **Smoke test rationale**:
   - The `cargo test --workspace --release` step already runs all integration tests (including dlp-e2e) against release builds.
   - The additional "dlp-server health check" smoke test verifies the actual release binary can start and serve requests, which is a stronger guarantee than in-process router tests.
   - The health check uses the default bind address (127.0.0.1:9090). If the server uses a different default, adjust the URL.

5. **Notification** (optional, documented but not implemented):
   - The workflow does NOT include Slack/email notifications. These can be added later if needed.
   - Failed nightly runs are visible in the GitHub Actions tab.
