Update `.github/workflows/build.yml` to add a new `test` job:

1. Add a new job `test` at the same level as `sonarqube`:
   ```yaml
   test:
     name: Test Workspace
     runs-on: windows-latest
     needs: []  # Run in parallel with SonarQube; no dependency
     steps:
       - uses: actions/checkout@v4

       - name: Install Rust toolchain
         uses: dtolnay/rust-action@stable

       - name: Cache cargo registry
         uses: actions/cache@v4
         with:
           path: |
             ~/.cargo/registry
             ~/.cargo/git
             target
           key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}

       - name: Build workspace (zero warnings)
         run: cargo build --workspace
         env:
           RUSTFLAGS: "-D warnings"

       - name: Run clippy
         run: cargo clippy --workspace -- -D warnings

       - name: Check formatting
         run: cargo fmt --check

       - name: Run tests
         run: cargo test --workspace
         env:
           RUSTFLAGS: "-D warnings"
   ```

2. The job MUST:
   - Use `runs-on: windows-latest` (D-13)
   - Set `RUSTFLAGS: "-D warnings"` for both build and test steps (D-21)
   - Run `cargo clippy --workspace -- -D warnings` as a separate step
   - Run `cargo fmt --check` as a separate step
   - Run `cargo test --workspace` (D-12)
   - NOT set `CARGO_TARGET_DIR` (D-14 — default target is fine in CI)
   - Use `dtolnay/rust-action@stable` for Rust toolchain installation (per 30-RESEARCH.md State of the Art)
   - Include cargo cache for faster builds

3. The `test` job should run in parallel with `sonarqube` (no `needs:` dependency) so PR checks complete faster.

4. Preserve the existing `sonarqube` job exactly as-is.
