## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI client -> Admin API | Policy payload crosses from local TUI to server; JWT required |
| User input -> form fields | Free-text name/description/priority from keyboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | PolicyPayload conditions JSON | mitigate | Conditions serialized from typed `Vec<PolicyCondition>` via serde — no raw JSON entry in TUI (per RESEARCH.md anti-pattern). Server also validates condition structure on ingest. |
| T-14-02 | Elevation of Privilege | POST /admin/policies endpoint | mitigate | Server middleware requires valid admin JWT (AdminUsername::extract_from_headers). TUI sends JWT from authenticated session. No change needed — already enforced. |
| T-14-03 | Spoofing | Policy ID generation | mitigate | UUID v4 generated via `uuid::Uuid::new_v4()` — cryptographically random, collision-resistant. Server enforces UNIQUE constraint on id column. |
| T-14-04 | Information Disclosure | Validation error display | accept | Error messages from server may include internal details (e.g., "UNIQUE constraint failed"). Acceptable for admin-only TUI — no public exposure. |
| T-14-05 | Denial of Service | Priority field as u32 | mitigate | Parsed as u32 client-side; rejects negative values and non-numeric input before POST. Server also stores as INTEGER. |
</threat_model>
