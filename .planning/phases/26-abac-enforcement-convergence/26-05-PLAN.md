Required observable behaviors (D-08, D-09) — must all be covered by named tests:

**D-09 drive-letter extraction edge cases:**
- Path `"e:\\file.txt"` (lowercase drive letter) → drive letter extracted as `'E'` (uppercase)
- Path `"\\\\server\\share\\file.txt"` (UNC) → `None` immediately
- Path `""` (empty) → `None`
- Path `"/linux/style/path"` (non-alpha first char) → `None`

**D-08 write-class variants on ReadOnly:**
- `FileAction::Written` → `Some(Decision::DENY)`
- `FileAction::Created` → `Some(Decision::DENY)`
- `FileAction::Deleted` → `Some(Decision::DENY)`
- `FileAction::Moved` → `Some(Decision::DENY)` (this is the "Renamed" case from D-08)
- `FileAction::Read` → `None` (allowed)

**D-07 Blocked tier:**
- All five FileAction variants → `Some(Decision::DENY)` (including Read)

**D-07 FullAccess tier:**
- All five FileAction variants → `None`

**Drive not in device_identities (not a USB drive):**
- Path on drive letter not present in `UsbDetector::device_identities` → `None`

**Default-blocked unknown device (fail-safe):**
- DeviceIdentity exists for drive letter, but VID/PID/serial NOT in registry → `DeviceRegistryCache::trust_tier_for` returns `Blocked` (default) → `Some(Decision::DENY)`

}
