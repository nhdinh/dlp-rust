**Gathered:** 2026-04-11

<domain>
## Phase Boundary

Comprehensive test suite for all detection modules, server-side evaluation, and end-to-end
pipeline coverage. Three waves, extending existing test files:

**Wave 1 — Detection module unit tests** (`dlp-agent/tests/comprehensive.rs`):
- USB detection: `usb.rs` — GetDriveTypeW, device arrival/removal, blocked-drive set
- Network share detection: `network_share.rs` — WNetOpenEnumW, SmbShareEvent, differential scan
- Clipboard classification: `clipboard/` — classify_text, T4/T3 detection, edge cases
- File monitor: `file_monitor.rs` — built-in exclusions, excluded path matching
- `#[cfg(windows)]` guards — tests compile everywhere, run only on Windows

**Wave 2 — Server-side evaluation tests** (`dlp-server/tests/` or new module):
- Policy CRUD: create/update/delete + validation
- `POST /audit/events`: round-trip AuditEvent storage, queryability
- Policy evaluation: `POST /audit/events` with various decisions (ALLOW, DENY, DenyWithAlert)
- Auth: JWT-protected routes reject unauthenticated requests
- Existing dlp-server lib tests extended

**Wave 3 — End-to-end pipeline integration** (`dlp-agent/tests/integration.rs`):
- File action → classify → policy eval → block/allow → audit event emitted
- Clipboard paste → classify_text → DenyWithAlert → audit event
- SMB share detection → SmbShareEvent → policy eval → audit event
- USB write → blocked → audit event
- Error paths: engine unreachable, 429, cache miss T4

Out of scope:
- Load/stress testing
- Integration with real AD (Phase 7)
- Multi-agent scenarios

</domain>

<decisions>
## Implementation Decisions

### Test location — extend existing test files
- **D-01:** Wave 1 (detection unit tests) added to `dlp-agent/tests/comprehensive.rs`
- **D-02:** Wave 3 (E2E pipeline) added to `dlp-agent/tests/integration.rs`
- **D-03:** Wave 2 (server evaluation) in `dlp-server/tests/` or inline `#[cfg(test)]` modules

### Windows API strategy — real APIs, windows-only tests
- **D-04:** Detection module tests use `#[cfg(windows)]` so they compile on all platforms but only run on Windows. This is appropriate — DLP-Rust is a Windows-only product.
- **D-05:** Windows API calls (GetDriveTypeW, WNetOpenEnumW, RegisterDeviceNotificationW) are real calls inside `#[cfg(windows)]` blocks in tests, not mocked.

### Test organization
- **D-06:** Each detection module gets its own `mod detection_tests` block inside the parent test file
- **D-07:** Mock engines (in-process axum servers) for E2E and server tests — no external dependencies
- **D-08:** `cargo test --workspace` passes on Windows (tests with `#[cfg(windows)]` are skipped on non-Windows)

### Folded Todos
None.

</decisions>
