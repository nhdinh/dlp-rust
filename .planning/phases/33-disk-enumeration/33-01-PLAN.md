9. **Windows implementation `enumerate_fixed_disks_windows()`**:
   - Use `SetupDiGetClassDevsW(Some(&GUID_DEVINTERFACE_DISK), PCWSTR::null(), None, DIGCF_DEVICEINTERFACE | DIGCF_PRESENT)` to enumerate disk devices.
   - Loop with `SetupDiEnumDeviceInfo` (safety valve at 1024 iterations).
   - For each disk device:
     a. Read instance ID via `SetupDiGetDeviceInstanceIdW`.
     b. Read model via `read_string_property(hdev, &devinfo, SPDRP_FRIENDLYNAME).or_else(|| read_string_property(..., SPDRP_DEVICEDESC))`.
     c. Determine drive letter: scan A..=Z, call `GetDriveTypeW` for each. A drive is DRIVE_FIXED (value 3) and maps to this disk if the instance ID matches. For now, use a simplified heuristic: enumerate all fixed drives via `GetLogicalDrives` + `GetDriveTypeW`, then correlate by volume. Use `std::path::Path::new(&format!("{}:\\", letter)).exists()` as secondary check per D-05/31-02 pattern.
     d. Query bus type via `query_bus_type_ioctl(&device_interface_path)` (primary) + `is_usb_bridged_pnp_walk(&instance_id)` (fallback).
     e. Build `DiskIdentity` with all fields. Set `size_bytes: None` for now (WMI query deferred to Phase 34).
   - Call `SetupDiDestroyDeviceInfoList(hdev)` before returning.
   - Boot disk detection: call `get_boot_drive_letter_windows()` to get the system drive letter, then set `is_boot_disk = true` on the matching disk.

10. **Windows implementation `query_bus_type_ioctl(device_interface_path: &str) -> Result<BusType, DiskError>`**:
    - Construct a device path from instance ID for CreateFileW. Use `\\.\\PhysicalDriveN` format. To map instance_id to PhysicalDrive index, use a helper that enumerates `Win32_DiskDrive` WMI and matches `PNPDeviceID` to instance_id.
    - Alternative simpler approach: Open the device via `CreateFileW` with `\\.\\PhysicalDriveN` where N is derived from the disk enumeration order (0, 1, 2...). The SetupDi enumeration order typically matches PhysicalDrive numbering.
    - Actually, per D-12/D-13 and CONTEXT.md: use `IOCTL_STORAGE_QUERY_PROPERTY` with `StorageDeviceProperty`. The `STORAGE_DEVICE_DESCRIPTOR` contains `BusType`.
    - Send `DeviceIoControl` with `IOCTL_STORAGE_QUERY_PROPERTY`, pass a `STORAGE_PROPERTY_QUERY` with `PropertyId = StorageDeviceProperty` (0), `QueryType = PropertyStandardQuery` (0).
    - Read `STORAGE_DEVICE_DESCRIPTOR.BusType` field (offset varies; use the `RawProperties` byte array and parse from offset). The Windows crate 0.61 may have different struct layout than 0.58. Use a manual byte-parse approach: the descriptor starts with `STORAGE_DEVICE_DESCRIPTOR` header, then `RawProperties` follows. The `BusType` is at a fixed offset in the struct.
    - If DeviceIoControl fails, return Err(DiskError::IoctlFailed(...)).

11. **Windows implementation `is_usb_bridged_windows(instance_id: &str) -> Result<bool, DiskError>`**:
    - First try `query_bus_type_ioctl(instance_id)` — if it returns `BusType::Usb`, return `Ok(true)`.
    - If IOCTL fails or returns non-USB, try PnP tree walk: `CM_Locate_DevNodeW` -> walk up with `CM_Get_Parent` / `CM_Get_Device_IDW` for up to 16 levels. If any ancestor starts with `USB\`, return `Ok(true)`.
    - If neither method indicates USB, return `Ok(false)`.

12. **Windows implementation `is_usb_bridged_pnp_walk(instance_id: &str) -> Result<bool, DiskError>`**:
    - Same PnP tree walk logic as in usb.rs `on_disk_device_arrival`.
    - Return true if USB ancestor found.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kernel -> User (IOCTL) | Untrusted kernel response to IOCTL_STORAGE_QUERY_PROPERTY could be manipulated by compromised driver |
| WMI -> Agent | Win32_DiskDrive WMI data is OS-provided; attacker with admin could spoof |
| Audit Event -> SIEM | Disk identity data in audit events must not contain sensitive PII |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | IOCTL_STORAGE_QUERY_PROPERTY response | mitigate | BusType is cross-validated with PnP tree walk (D-13/D-14); single-source failure does not allow USB-bridged disk to masquerade as internal |
| T-33-02 | Information Disclosure | Disk serial number in audit events | accept | Serial numbers are hardware identifiers, not user PII; required for allowlist management per D-10 |
| T-33-03 | Denial of Service | SetupDi enumeration hangs on corrupted device tree | mitigate | All enumeration loops have 1024-iteration safety valves; IOCTL has timeouts via CreateFileW flags |
| T-33-04 | Elevation of Privilege | Boot disk misidentification allows blocking system drive | mitigate | Boot disk detection uses GetSystemDirectoryW (OS-resolved) plus cross-reference with enumerated disks; is_boot_disk flag is set at enumeration time and never user-modifiable |
</threat_model>
