<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet / LAN admin client → /admin/disk-registry | Untrusted JSON request body crosses into the server. JWT middleware authenticates the principal; per-handler validation authorizes the payload. |
| Admin handler → SQLite | Validated payloads cross into persistent storage via UnitOfWork. CHECK + UNIQUE constraints are the second line of defense after handler validation. |
| Server → polling agent | The `disk_allowlist` field returned by `GET /agent-config/{id}` propagates allowlist changes. The endpoint is public-route (agents do not authenticate via JWT) — same security posture as the existing USB device-registry public read. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-04 | Spoofing / Elevation | unauthenticated POST/DELETE/GET on /admin/disk-registry | mitigate | All three handlers wired into `protected_routes` (line 552 of admin_api.rs); `admin_auth::require_auth` middleware enforces JWT; `AdminUsername::extract_from_headers` re-verifies the JWT inside the handler so the audit username is JWT-derived, not request-supplied. |
| T-37-05 | Tampering | injection via `encryption_status` payload field | mitigate | Length guard (>32 chars rejects with 422 BEFORE allocation); const VALID_STATUSES allowlist BEFORE DB access; DB CHECK constraint as defense in depth. Same belt-and-braces pattern used for `trust_tier` in upsert_device_registry_handler. |
| T-37-06 | Tampering / Repudiation | duplicate (agent_id, instance_id) "pre-register attack" — admin attempts to overwrite an existing entry's encryption_status by re-POSTing | mitigate | Pure INSERT (Plan 01); UNIQUE conflict surfaces as rusqlite extended_code 2067; handler maps to AppError::Conflict (409). Per D-05 admin must DELETE then re-add to change fields — no silent mutation. |
| T-37-07 | Repudiation / Audit Bypass | admin add/remove without audit event | mitigate | Audit emission is in a SECOND spawn_blocking AFTER the main commit (D-10). DB write commits first; audit commit is separate transaction so audit failure does NOT roll back. Both POST and DELETE handlers emit exactly one event. |
| T-37-08 | Information Disclosure | unauthenticated callers enumerate the fleet's disk allowlist | mitigate | GET /admin/disk-registry placed in `protected_routes` (NOT `public_routes`) per CONTEXT.md Claude's Discretion. The `disk_allowlist` exposure on the public `GET /agent-config/{id}` is restricted to entries scoped to the requested agent_id only — no cross-agent leak. |
| T-37-09 | Tampering | physical relocation attack — disk moved from machine-A to machine-B | mitigate | Schema enforces (agent_id, instance_id) scoping (Plan 01); admin allowlisting machine-A/disk-1 does NOT allowlist machine-B/disk-1. The `list_by_agent` filter in get_agent_config_for_agent ensures the relocated disk is NOT in machine-B's payload until an admin re-registers it for machine-B. |
</threat_model>

    /// `POST /admin/disk-registry` -- ADMIN-03 + AUDIT-03. JWT-protected.
    ///
    /// Pure INSERT (D-05). Returns 409 Conflict on duplicate `(agent_id, instance_id)`.
    /// Returns 422 if `encryption_status` is outside the four-value allowlist.
    /// Emits an `AdminAction(DiskRegistryAdd)` audit event AFTER the registry commit (D-10).
    async fn insert_disk_registry_handler(
        State(state): State<Arc<AppState>>,
        req: axum::http::Request<axum::body::Body>,
    ) -> Result<(StatusCode, Json<DiskRegistryResponse>), AppError> {
        // (1) Authenticate the admin via JWT extraction (T-37-04).
        let username = AdminUsername::extract_from_headers(req.headers())?;

        // (2) Deserialize the body. Must follow username extraction so the body
        //     is consumed last and the headers are available before the body move.
        let Json(body) = Json::<DiskRegistryRequest>::from_request(req, &state)
            .await
            .map_err(AppError::from)?;

        // (3) Length guard before heap allocation (T-37-05). Valid values are at
        //     most 21 chars ("partially_encrypted"); 32 is a generous ceiling.
        if body.encryption_status.len() > 32 {
            return Err(AppError::UnprocessableEntity(
                "encryption_status exceeds maximum length".to_string(),
            ));
        }
        // (4) Allowlist check before any DB access (D-12, T-37-05).
        const VALID_STATUSES: &[&str] = &[
            "fully_encrypted",
            "partially_encrypted",
            "unencrypted",
            "unknown",
        ];
        if !VALID_STATUSES.contains(&body.encryption_status.as_str()) {
            return Err(AppError::UnprocessableEntity(format!(
                "invalid encryption_status '{}'; must be one of: fully_encrypted, \
                 partially_encrypted, unencrypted, unknown",
                body.encryption_status
            )));
        }

        let row = DiskRegistryRow {
            id: id.clone(),
            agent_id: body.agent_id.clone(),
            instance_id: body.instance_id.clone(),
            bus_type: body.bus_type.clone(),
            encryption_status: body.encryption_status.clone(),
            model: body.model.clone(),
            registered_at: registered_at.clone(),
        };

        let pool = Arc::clone(&state.pool);

                        return AppError::Conflict(format!(
                            "disk (agent_id={}, instance_id={}) already registered",
                            row_for_insert.agent_id, row_for_insert.instance_id
                        ));
                    }
                }
                AppError::Database(e)
            })?;
            uow.commit().map_err(AppError::Database)?;
            Ok(())
        })
        .await
        .map_err(|e| AppError::Internal(anyhow::anyhow!("join error: {e}")))??;

        //     Audit failure must NOT roll back the registry write.
        let audit_event = dlp_common::AuditEvent::new(
            dlp_common::EventType::AdminAction,
            String::new(),
            username,
            format!("disk:{}@{}", body.instance_id, body.agent_id),
            dlp_common::Classification::T3,
            dlp_common::Action::DiskRegistryAdd,
            dlp_common::Decision::ALLOW,
            "server".to_string(),
            0,
        );

        tokio::task::spawn_blocking(move || -> Result<_, AppError> {
            let mut conn = pool.get().map_err(AppError::from)?;

    Step 2 — `delete_disk_registry_handler`. Two-conn pattern: first read `(agent_id, instance_id)` for the audit resource, then DELETE inside the same spawn_blocking (with explicit conn scope to avoid pool deadlock):

    ```rust
    /// `DELETE /admin/disk-registry/{id}` -- ADMIN-03 + AUDIT-03. JWT-protected.
    ///
    /// Returns 204 on success, 404 if the UUID does not exist. Emits
    /// `AdminAction(DiskRegistryRemove)` AFTER the delete commit (D-10).
    async fn delete_disk_registry_handler(
        State(state): State<Arc<AppState>>,
        req: axum::http::Request<axum::body::Body>,
    ) -> Result<StatusCode, AppError> {
        // (1) Authenticate.
        let username = AdminUsername::extract_from_headers(req.headers())?;
        // (2) Extract path param.
        let id = Path::<String>::from_request(req, &state)
            .await
            .map_err(AppError::from)?
            .0;

        // (3) Read agent_id/instance_id (for audit resource) THEN delete in one
        //     spawn_blocking. Use explicit conn scope to release the read conn
        //     before re-acquiring for write (avoids max_size=1 pool deadlock).
        let pool = Arc::clone(&state.pool);
        let disk_id = id.clone();
        let result = tokio::task::spawn_blocking(
            move || -> Result<Option<(String, String, usize)>, AppError> {
                let (agent_id, instance_id) = {
                    let conn = pool.get().map_err(AppError::from)?;
                    match conn.query_row(
                        "SELECT agent_id, instance_id FROM disk_registry WHERE id = ?1",
                        rusqlite::params![disk_id],
                        |r| Ok((r.get::<_, String>(0)?, r.get::<_, String>(1)?)),
                    ) {
                        Ok(t) => t,
                        Err(rusqlite::Error::QueryReturnedNoRows) => return Ok(None),
                        Err(e) => return Err(AppError::Database(e)),
                    }
                }; // read conn dropped here -- returned to pool
                let mut conn = pool.get().map_err(AppError::from)?;
                let uow = db::UnitOfWork::new(&mut conn).map_err(AppError::Database)?;
                let n = DiskRegistryRepository::delete_by_id(&uow, &disk_id)
                    .map_err(AppError::Database)?;
                uow.commit().map_err(AppError::Database)?;
                Ok(Some((agent_id, instance_id, n)))
            },
        )
        .await
        .map_err(|e| AppError::Internal(anyhow::anyhow!("join error: {e}")))??;

        let (agent_id_for_audit, instance_id_for_audit, rows_deleted) = match result {
            Some(t) => t,
            None => return Err(AppError::NotFound(format!("disk entry {id} not found"))),
        };
        if rows_deleted == 0 {
            // Race: row existed at SELECT but was deleted by another caller before our DELETE.
            return Err(AppError::NotFound(format!("disk entry {id} not found")));
        }

        // (4) Second spawn_blocking: audit emission (D-10, T-37-07).
        let audit_event = dlp_common::AuditEvent::new(
            dlp_common::EventType::AdminAction,
            String::new(),
            username,
            format!("disk:{}@{}", instance_id_for_audit, agent_id_for_audit),
            dlp_common::Classification::T3,
            dlp_common::Action::DiskRegistryRemove,
            dlp_common::Decision::ALLOW,
            "server".to_string(),
            0,
        );
        let pool = Arc::clone(&state.pool);
        tokio::task::spawn_blocking(move || -> Result<_, AppError> {
            let mut conn = pool.get().map_err(AppError::from)?;
            let uow = db::UnitOfWork::new(&mut conn).map_err(AppError::Database)?;
            audit_store::store_events_sync(&uow, &[audit_event])?;
            uow.commit().map_err(AppError::Database)?;
            Ok(())
        })
        .await
        .map_err(|e| AppError::Internal(anyhow::anyhow!("join error: {e}")))??;

        tracing::info!(
            agent_id = %agent_id_for_audit,
            instance_id = %instance_id_for_audit,
            "disk registry remove"
        );
        Ok(StatusCode::NO_CONTENT)
    }
    ```
