```rust
/// Parses an agent config row from a `rusqlite::Row`.
///
/// Column order must be: monitored_paths (TEXT), heartbeat_interval_secs (INTEGER),
/// offline_cache_enabled (INTEGER).
fn parse_agent_config_row(row: &rusqlite::Row) -> rusqlite::Result<AgentConfigPayload> {
    let paths_json: String = row.get(0)?;
    let monitored_paths: Vec<String> = serde_json::from_str(&paths_json)
        .unwrap_or_default();
    let heartbeat_interval_secs = u64::try_from(row.get::<_, i64>(1)?)
        .unwrap_or(30);
    let offline_cache_enabled = row.get::<_, i64>(2)? != 0;
    Ok(AgentConfigPayload {
        monitored_paths,
        heartbeat_interval_secs,
        offline_cache_enabled,
    })
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent -> GET /agent-config/{id} | Unauthenticated endpoint; any caller with an agent_id can read that agent's config |
| Admin -> PUT/DELETE /admin/agent-config/* | JWT-protected; only authenticated admin can write config |
| Admin input -> DB | monitored_paths and heartbeat_interval_secs are user-controlled input stored in DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | GET /agent-config/{id} | accept | Endpoint is deliberately unauthenticated per CONTEXT.md. Config contains no secrets (paths, interval, bool). An attacker who knows an agent_id can read its config but gains no escalation. |
| T-06-02 | Tampering | PUT /admin/agent-config | mitigate | JWT required via require_auth middleware on all PUT/DELETE routes. Unauthenticated PUT returns 401. |
| T-06-03 | Denial of Service | heartbeat_interval_secs = 0 | mitigate | Validate heartbeat_interval_secs >= 10 at PUT time; reject with 400 BadRequest if < 10. |
| T-06-04 | Tampering | monitored_paths injection | accept | Server stores paths verbatim (admin-controlled). Agent uses paths only for local fs watch, not for server-side operations. No path traversal risk on server. Low risk. |
| T-06-05 | Information Disclosure | Agent config contents | accept | Config contains only directory paths, an interval integer, and a boolean. No secrets, tokens, or PII. Safe to serve without auth. |
</threat_model>
