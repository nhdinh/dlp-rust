<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UsbBlockResult → toast body | Device description from hardware registry crosses into user-visible notification text |
| BROADCASTER.broadcast | Fire-and-forget write to named pipe; no response expected |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-04 | Information Disclosure | identity.description in toast | accept | Hardware device description is not sensitive PII; shown only to the local session user who triggered the block |
| T-27-05 | Tampering | Pipe2AgentMsg::Toast payload | accept | Pipe 2 is a local named pipe in a SYSTEM service context; accessible only to processes with appropriate Windows ACLs already enforced by the OS |
| T-27-06 | Denial of Service | BROADCASTER.broadcast fire-and-forget | mitigate | broadcast() drops frames silently when client queue is full (CLIENT_QUEUE_DEPTH=64); no blocking possible |
| T-27-07 | Spoofing | unreachable! arm for FullAccess | mitigate | Compile-time exhaustive match ensures FullAccess cannot silently produce a toast; unreachable!() panics in debug and is elided in release via optimizer |
</threat_model>
