## Accomplishments

- Added `Screen::LdapConfig { .. } => handle_ldap_config(app, key)` routing arm in `handle_event`.
- Expanded `handle_system_menu` from 5 to 6 items: row 4 dispatches to `action_load_ldap_config`, row 5 returns to `MainMenu`.
- Added `LDAP_KEYS` constant array with the 5 `LdapConfigPayload` field names in exact server serialization order.
- Added `ldap_is_bool()` (row 2) and `ldap_is_numeric()` (row 3) helper predicates.
- Implemented `action_load_ldap_config()` — GET `admin/ldap-config`, stores response into `Screen::LdapConfig`.
- Implemented `action_save_ldap_config()` — PUT `admin/ldap-config`, on success returns to `SystemMenu { selected: 4 }`.
- Implemented `handle_ldap_config()` dispatcher delegating to editing or nav based on `editing` flag.
- Implemented `handle_ldap_config_editing()` with full text/numeric entry: `cache_ttl_secs` parses as `u64`, validates `[60, 3600]`, stores as `serde_json::Value::Number`, rejects with `"Cache TTL must be between 60 and 3600 seconds"` and stays in edit mode.
- Implemented `handle_ldap_config_nav()` with Up/Down wraparound, Save (row 5), Back (row 6), bool toggle (row 2), numeric pre-fill (row 3), string pre-fill (rows 0,1,4), and Esc returning to `SystemMenu { selected: 4 }`.
- Added `ldap_config_tests` module with 4 unit tests verifying key order, row constants, bool predicate, and numeric predicate.
- Verified `cargo check --package dlp-admin-cli` produces exactly one error: the expected non-exhaustive match in `render.rs` (to be closed by plan 38.1-03).
