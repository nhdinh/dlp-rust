<decisions>
## Implementation Decisions

### Pattern: Follow SIEM/Alert Config Screen Exactly
- Mirror the exact pattern from `Screen::SiemConfig` and `Screen::AlertConfig` in Phase 28
- Same Screen variant shape: `{ config: serde_json::Value, selected: usize, editing: bool, buffer: String }`
- Same dispatch split: `handle_ldap_config_editing()` + `handle_ldap_config_nav()` called from `handle_ldap_config()`
- Same render pattern: constant label array + JSON key array + bool/numeric helper fns + `draw_ldap_config()`

### Fields (5 editable + 2 action rows = 7 total rows)
- Index 0: `ldap_url` — text input, default `(empty)`
- Index 1: `base_dn` — text input, default `(empty)`
- Index 2: `require_tls` — boolean toggle (`[x]` / `[ ]`), maps to `bool` JSON
- Index 3: `cache_ttl_secs` — numeric input, validates `60..=3600`, maps to `u64` JSON Number
- Index 4: `vpn_subnets` — text input, comma-separated CIDRs, default `(empty)`
- Index 5: `[ Save ]` — calls `action_save_ldap_config()`
- Index 6: `[ Back ]` — returns to `Screen::SystemMenu { selected: 4 }`

### Numeric Validation
- `cache_ttl_secs` must parse as `u64` in range `[60, 3600]`
- On failure: display error `"Cache TTL must be between 60 and 3600 seconds"`, stay in edit mode
- On success: store as `serde_json::Value::Number(serde_json::Number::from(ttl))`

### System Menu Integration
- Current SystemMenu has 5 items (indices 0..=4): Server Status, Agent List, SIEM Config, Alert Config, Back
- Expand to 6 items: insert "LDAP Config" at index 4, move "Back" to index 5
- Navigation wraparound: update `nav(selected, 5, ...)` to `nav(selected, 6, ...)`
- Label constant in render.rs: add `"LDAP Config"` at position 4

### Client Methods
- Use `app.client.get::<serde_json::Value>("admin/ldap-config")` directly in dispatch action fns
- Use `app.client.put::<serde_json::Value, _>("admin/ldap-config", &payload)` for save
- This is the actual pattern used by ALL existing screens (no named wrapper methods exist in `client.rs`);
  no `client.rs` modifications are required

### No Server Changes
- All server endpoints exist; `dlp-server` is untouched
- No new routes, no migration, no new structs needed

### Claude's Discretion
- Exact menu label string: "LDAP Config" (matches field naming convention)
- `SystemMenu { selected: 4 }` is the index for "Back" after expansion; save returns user to index 4 (LDAP Config row) not index 5 (Back row)

</decisions>
