## Key Changes

### `dlp-admin-cli/src/screens/render.rs`

- **SystemMenu expansion:** Added `"LDAP Config"` at index 4 between `"Alert Config"` and `"Back"`, expanding the menu from 5 to 6 items.
- **`draw_screen` routing:** Added `Screen::LdapConfig` match arm that delegates to `draw_ldap_config(frame, area, config, *selected, *editing, buffer)`.
- **`LDAP_FIELD_LABELS`:** New constant `[&str; 7]` with display labels for all 7 rows: `LDAP URL`, `Base DN`, `Require TLS`, `Cache TTL (secs)`, `VPN Subnets`, `[ Save ]`, `[ Back ]`.
- **`is_ldap_bool`:** Returns `true` only for index 2 (`require_tls` boolean toggle row).
- **`is_ldap_numeric`:** Returns `true` only for index 3 (`cache_ttl_secs` numeric row).
- **`draw_ldap_config`:** Full render function mirroring the SIEM/Alert config pattern:
  - Maps row indices to JSON keys (`ldap_url`, `base_dn`, `require_tls`, `cache_ttl_secs`, `vpn_subnets`).
  - Boolean row renders as `[x]` or `[ ]`.
  - Numeric row renders the `u64` value, defaulting to 300 if absent.
  - Text rows render the string value or `(empty)` if blank.
  - Edit mode shows the buffer with a trailing `_` cursor marker.
  - Action rows (`[ Save ]`, `[ Back ]`) render as plain labels.
  - Uses the standard cyan highlight style with `> ` symbol.
  - Contextual hints bar: edit mode vs navigation mode.
- **`ldap_render_tests` module:** 3 unit tests verifying label count/order, bool predicate correctness, and numeric predicate correctness.
