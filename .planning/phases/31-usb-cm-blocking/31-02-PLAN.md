<objective>
Fix USB device arrival detection so that Blocked-tier USB mass storage devices are reliably disabled via CM_Disable_DevNode, even when Windows does not fire the GUID_DEVINTERFACE_USB_DEVICE notification.

Purpose: The current code only registers for GUID_DEVINTERFACE_VOLUME (drive letter tracking) and GUID_DEVINTERFACE_USB_DEVICE (VID/PID/serial capture + tier enforcement). Many USB mass storage devices never fire the USB_DEVICE notification, so the enforcement path (apply_tier_enforcement -> DeviceController::disable_usb_device) is never triggered. This gap caused Test 6 to fail: the device remained visible and writable in Windows Explorer.

Output: Updated dlp-agent/src/detection/usb.rs with GUID_DEVINTERFACE_DISK registration and a PnP tree-walk handler that finds the USB ancestor, parses VID/PID/serial, and calls apply_tier_enforcement — matching the approach already proven correct in Phase 32's dlp-common/src/usb.rs.
</objective>

### From dlp-common/src/usb.rs (lines 267-351) — enumerate_connected_usb_devices_windows

This function shows the correct PnP tree walk pattern:

```rust
// Walk up the PnP device tree to find a USB ancestor.
let mut usb_identity: Option<DeviceIdentity> = None;
let mut current_devinst = devinfo.DevInst;
for _ in 0..16 {
    let mut parent_devinst: u32 = 0;
    let cr = unsafe { CM_Get_Parent(&mut parent_devinst, current_devinst, 0) };
    if cr.0 != 0 {
        break;
    }

    let mut id_buf = [0u16; 256];
    let cr = unsafe { CM_Get_Device_IDW(parent_devinst, &mut id_buf, 0) };
    if cr.0 == 0 {
        let id = String::from_utf16_lossy(
            &id_buf.iter().copied().take_while(|&w| w != 0).collect::<Vec<u16>>(),
        );
        if id.starts_with("USB\\") {
            let reshaped = format!("\\\\?\\{}", id.replace('\\', "#"));
            let identity = parse_usb_device_path(&reshaped);
            if !identity.vid.is_empty() && !identity.pid.is_empty() {
                usb_identity = Some(identity);
            }
            break;
        }
    }
    current_devinst = parent_devinst;
}
```

|----------|-------------|
| OS PnP tree -> Agent | Untrusted input: device instance IDs from CM_Get_Device_IDW must not be parsed with assumptions about format beyond the `USB\` prefix check. |
| Agent -> CM APIs | The agent calls CM_Disable_DevNode with device instance handles derived from PnP tree walk — incorrect handles could disable wrong devices. |
