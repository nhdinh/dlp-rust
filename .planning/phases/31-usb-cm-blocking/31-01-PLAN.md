# Phase 31 Plan 01: USB CM Device Blocking

**Replace passive `notify`-based USB enforcement with active PnP device control using Windows CM_* APIs, and fix toast notification by adding UI binary startup check.**

## Wave 1: Device Controller Core

### Task 1: Create device_controller.rs with CM_* APIs

**Files:** `dlp-agent/src/device_controller.rs` (new)

**read_first:**
- `dlp-agent/src/detection/usb.rs` — understand VID/PID/serial format from dbcc_name
- `dlp-agent/src/usb_enforcer.rs` — understand trust tier evaluation
- Windows SDK docs for CM_Locate_DevNodeW, CM_Disable_DevNode, CM_Enable_DevNode

**action:**
Create `dlp-agent/src/device_controller.rs` with:

1. **Imports**: `windows::Win32::Devices::DeviceAndDriverInstallation::*`, `windows::Win32::Security::*`, `parking_lot::Mutex`, `std::collections::HashMap`

2. **`DeviceController` struct:**
```rust
pub struct DeviceController {
    /// Original DACLs keyed by drive letter, for restoration on removal.
    original_dacls: Mutex<HashMap<char, Vec<u8>>>,
}
```

3. **`disable_usb_device(vid, pid, serial)`**:
   - Build device instance ID from VID/PID/serial: `USB\VID_XXXX&PID_YYYY\SERIAL`
   - Call `CM_Locate_DevNodeW` with `CM_LOCATE_DEVNODE_NORMAL`
   - Call `CM_Disable_DevNode` with `CM_DISABLE_ABSOLUTE` (0) or `CM_DISABLE_HARDWARE` (1)
   - Return `Ok(())` on success, `Err` with descriptive message on failure
   - **Fail-safe**: If device not found, log warning but do NOT return error (device may have been removed between scan and enforcement)

4. **`enable_usb_device(vid, pid, serial)`**:
   - Same locate logic as disable
   - Call `CM_Enable_DevNode` with `CM_ENABLE_ABSOLUTE` (0)
   - Return `Ok(())` on success

5. **`set_volume_readonly(drive_letter)`**:
   - Open volume root: `\\.\X:` where X is the drive letter
   - Query existing DACL via `GetFileSecurity` with `DACL_SECURITY_INFORMATION`
   - Clone DACL bytes into `original_dacls` cache keyed by drive letter
   - Create new DACL: copy all existing ACEs except WRITE/DELETE for Everyone / Authenticated Users
   - Apply via `SetFileSecurity`
   - Close handle

6. **`restore_volume_acl(drive_letter)`**:
   - Look up original DACL bytes from `original_dacls`
   - If found: apply via `SetFileSecurity`, remove from cache
   - If not found: log warning (device may have been removed before we could restore)

7. **`new()` constructor** returning empty `DeviceController`

8. **Unit tests** (TDD):
   - `test_device_controller_new_empty_cache`
   - `test_dacl_backup_roundtrip` — mock the Win32 calls if possible, or test the HashMap logic directly

### Task 2: Wire device controller into usb_wndproc

**Files:** `dlp-agent/src/detection/usb.rs`

**read_first:**
- `dlp-agent/src/detection/usb.rs` — `usb_wndproc`, `on_usb_device_arrival`, `on_usb_device_removal`
- `dlp-agent/src/device_controller.rs` — functions created in Task 1

**action:**
1. Add `use crate::device_controller::DeviceController;`
2. Add `use crate::device_registry::DeviceRegistryCache;`
3. Add static `DEVICE_CONTROLLER: OnceLock<Arc<DeviceController>> = OnceLock::new();`
4. Add pub fn `set_device_controller(controller: Arc<DeviceController>)` that sets the static
5. In `on_usb_device_arrival` (after identity is captured and drive letter resolved):
   - Look up trust tier from registry cache using VID/PID/serial
   - If `Blocked`: call `DEVICE_CONTROLLER.get().unwrap().disable_usb_device(vid, pid, serial)`
   - If `ReadOnly`: call `DEVICE_CONTROLLER.get().unwrap().set_volume_readonly(letter)`
   - If `FullAccess`: do nothing (device stays enabled)
   - Log result at INFO level
6. In `on_usb_device_removal` (after clearing device_identities):
   - Call `DEVICE_CONTROLLER.get().unwrap().restore_volume_acl(letter)` for the removed drive letter
   - Call `DEVICE_CONTROLLER.get().unwrap().enable_usb_device(vid, pid, serial)` if device was disabled
   - Log result at INFO level
7. Add `DEVICE_CONTROLLER` initialization in `service.rs` alongside `UsbEnforcer` construction

   - Remove `test_readonly_device_denies_write_class` (no longer applicable)
