## Phase 7: Active Directory LDAP Integration
**Requirement:** R-05
**Plans:** 3/3
**Completed:** 2026-04-16

**Files:** `dlp-common/src/ad_client.rs`, `dlp-common/src/abac.rs`, `dlp-agent/src/identity.rs`

**What was built:** LDAP client using `ldap3` crate. Queries AD for transitive group membership via `tokenGroups`, device trust via `NetGetJoinInformation`, and network location via `GetAdaptersAddresses` + VPN subnet matching. Machine account Kerberos TGT bind (no stored credentials). Results cached with configurable TTL.

**Key design decisions:**
- Channel-based async: AdClient spawns background Tokio task owning LDAP connection; mpsc + oneshot serializes LDAP ops
- Fail-open: empty groups on error, never block operations
- Group cache keyed by caller_sid (universally available); username used only for sAMAccountName LDAP filter

**UAT:** ABAC evaluation uses real AD group membership for policy decisions.

## Phase 8: Rate Limiting Middleware
**Requirement:** R-07
**Plans:** 1/1
**Completed:** 2026-04-15

**Files:** `dlp-server/src/rate_limiter.rs`, `dlp-server/Cargo.toml`, `dlp-server/src/lib.rs`, `dlp-server/src/admin_api.rs`, `dlp-server/src/main.rs`, `dlp-server/src/policy_api.rs`

**What was built:** `tower-governor` integration with 5 rate limit configs:
- `strict_config`: 5/min, IP key — `/auth/login`
- `moderate_config`: 30/min, agent-id — `/agents/{id}/heartbeat`
- `per_agent_config`: 200/min, agent-id — `/audit/events`
- `policy_config`: 60/min, IP — policy CRUD
- `default_config`: 100/min, IP — remaining protected routes

Also required axum 0.7 → 0.8 upgrade.

**UAT:** Rapid-fire login attempts are throttled with 429 responses including `Retry-After` header.

## Phase 9: Admin Operation Audit Logging
**Requirement:** R-09
**Plans:** 2/2
**Completed:** 2026-04-14

**Files:** `dlp-server/src/admin_api.rs`, `dlp-server/src/admin_auth.rs`, `dlp-server/src/audit_store.rs`, `dlp-server/src/lib.rs`, `dlp-server/src/db.rs`, `dlp-common/src/abac.rs`, `dlp-server/tests/admin_audit_integration.rs`

**What was built:** Structured `AuditEvent` records for policy CRUD and password changes persisted to `audit_events` with `EventType::AdminAction`. 4 integration tests verifying exact SQLite contents.

**Key fields:** `event_type=AdminAction`, `action_attempted=PolicyCreate|PolicyUpdate|PolicyDelete|PasswordChange`, `agent_id=server`, `classification=T3`, `decision=ALLOW`, `resource_path=policy:<id>` or `password_change:<username>`

**UAT:** Policy create/update/delete appear as audit events queryable via `GET /audit/events?event_type=ADMIN_ACTION`.

## Phase 10: SQLite Connection Pool
**Requirement:** R-10
**Plans:** 1/1
**Completed:** 2026-04-15

**Files:** `dlp-server/src/db.rs` (rewrite), all dlp-server handler files

**What was built:** `r2d2`/`r2d2_sqlite` connection pool replacing single `parking_lot::Mutex<Connection>`. `AppState { pool: db::Pool }` derives `Clone`. All handlers use `pool.get()` with proper `spawn_blocking` return types. `impl From<r2d2::Error>` on `AppError`, `SiemError`, `AlertError`.

**Verification:** 75 dlp-server lib tests + 145 dlp-agent lib tests pass, 0 clippy warnings.

**UAT:** Concurrent API requests execute without serializing on a single mutex.

## Phase 11: Policy Engine Separation
**Requirement:** R-03
**Plans:** 4/4 (waves)
**Completed:** 2026-04-16

**Files:** `dlp-server/src/policy_store.rs` (new), `dlp-server/src/policy_engine_error.rs` (new), `dlp-server/src/lib.rs`, `dlp-server/src/admin_api.rs`, `dlp-server/src/policy_api.rs`, `dlp-server/src/main.rs`, `dlp-server/src/db.rs`, `dlp-server/tests/policy_store_tests.rs`

**What was built:**
- `PolicyStore`: `parking_lot::RwLock<Vec<Policy>>` in-memory cache, sync hot-path `evaluate()`, tiered default-deny (T1/T2 → ALLOW, T3/T4 → DENY), 5 condition types, 23 unit tests
- `PolicyEngineError`: `thiserror`-derived enum with `PolicyNotFound` variant
- `POST /evaluate` wired into `public_routes` (unauthenticated, agent identity from `AgentInfo` body)
- Cache invalidation on every policy CRUD commit
- Background refresh via `tokio::time::interval` (5-minute interval)

**Wave breakdown:**
- Wave 1: Core types (`PolicyStore`, `PolicyEngineError`, unit tests)
- Wave 2: `AppState` wiring (`pool: db::Pool`, propagate to all handlers and test fixtures)
- Wave 3: `/evaluate` endpoint (`public_routes`, `evaluate_handler`, environment extraction)
- Wave 4: Testing (23 policy store tests, wave summaries)

**Known issue:** `:memory:` SQLite isolation bug in `test_invalidate_reloads_cache` and `test_refresh_reloads_cache` — connections in pool are isolated from test's connection.

**UAT:** Policy changes made via admin API take effect immediately (cache invalidated).
