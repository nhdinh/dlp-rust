# Milestone v0.6.0: Endpoint Hardening

**Status:** SHIPPED 2026-04-29
**Phases:** 22-30
**Total Plans:** 43

## Overview

Extend the enforcement layer with application identity, browser boundary control, and USB device control -- all surfaced as first-class ABAC subject attributes. Three parallel tracks: Application-aware DLP (SEED-001), Protected Clipboard browser boundary (SEED-002), and USB device-identity whitelist (SEED-003).

## Milestone Summary

**Key Decisions:**

- AppField enum defined in dlp-common/src/abac.rs -- policy DSL type, not identity type; placed before PolicyCondition to satisfy forward reference
- From<EvaluateRequest> for AbacContext drops agent field (tracing metadata, not ABAC attribute)
- AbacContext is the evaluation type from HTTP boundary inward -- no EvaluateRequest on PolicyStore::evaluate() hot path
- app_identity_matches fail-closed: Option<&AppIdentity> None returns false unconditionally
- UsbEnforcer check() fires before offline.evaluate() -- None return is zero-cost fast-path for non-USB drives; Some(DENY) short-circuits ABAC entirely
- extract_drive_letter normalizes to uppercase -- lowercase drive letters resolve to same HashMap key
- Toast broadcast additive before continue in USB block handler -- does not replace audit or BlockNotify
- Console mode passes None for usb_enforcer -- consistent with ad_client optional-subsystem pattern
- USB_DETECTOR static promoted to OnceLock<Arc<UsbDetector>> -- enables shared ownership with UsbEnforcer without cloning
- check() returns Option<UsbBlockResult> with notify flag -- rich result carries identity+tier+notify; block always DENY; notify gated by 30s per-drive cooldown
- Chrome pipe thread spawned in both service and console modes; shutdown sends origins_shutdown_tx and awaits poll handle
- Cross-platform compilation: removed #[cfg(windows)] from chrome/mod.rs so tests compile on all platforms
- Protobuf ContentAnalysisRequest/Response via prost; length-prefixed frame I/O with 4 MiB MAX_PAYLOAD cap
- HKLM self-registration at SOFTWARE\Google\Chrome\3rdparty\cas_agents; skipped when DLP_SKIP_CHROME_REG=1

**Issues Resolved:**

- All deferred UAT items from prior phases automated (Phase 4 webhook/SMTP, Phase 6 agent TOML write-back + hot-reload, Phase 24 release-mode build, Phase 28 all TUI screens)
- Pre-existing test_unregistered_device_defaults_to_read_only stale expectation fixed -- Zero Trust default-deny means Blocked, not ReadOnly
- Rustc STATUS_STACK_BUFFER_OVERRUN crash in integration test compilation resolved by using alternate target directory

**Issues Deferred:**

- SEED-002 Path A: Native browser extension (Chrome/Edge Manifest V3) -- deferred to post-v0.6.0
- APP-07: UWP app identity via AUMID -- deferred, needs spike
- USB-05: Audit device identity fields -- deferred
- USB-06: Per-user device registry -- deferred
- POLICY-F4, F5, F6: Export format, batch import, typed Decision -- deferred from v0.5.0

**Technical Debt Incurred:**

- Mount-time USB blocking requires kernel filter driver -- out of scope per CLAUDE.md
- Drag-and-drop interception -- different Win32 API surface, separate feature
- Edge for Business / Purview integration -- separate integration track
- Self-service device registration workflow -- IT-driven only in v0.6.0
