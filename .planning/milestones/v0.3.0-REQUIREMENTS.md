# REQUIREMENTS.md — v0.3.0 Operational Hardening

**Archived:** 2026-04-16 (shipped v0.3.0)

## Requirements

### R-01: SIEM Relay Integration
**Priority:** Must
**Status:** Validated
**Phase:** Phase 3 (v0.2.0)
**Acceptance:** When SIEM env vars are set, audit events appear in the configured Splunk/ELK instance.

### R-02: Alert Routing (Email + Webhook)
**Priority:** Must
**Status:** Validated
**Phase:** Phase 4 (v0.2.0)
**Acceptance:** After admin sets SMTP/webhook config via dlp-admin-cli, DenyWithAlert events trigger the configured email and/or webhook. Hot-reload on every send.

### R-03: Policy Engine Separation
**Priority:** Should
**Status:** Validated (adjusted)
**Phase:** Phase 11 (v0.3.0)
**Adjustment:** Full multi-replica sync (R-03 original description) deferred. v0.3.0 shipped PolicyStore with cache invalidation + background refresh. Replica sync remains future work.
**Acceptance:** In-memory policy cache hot-reloaded on CRUD. Background refresh every 5 minutes.

### R-04: Config Push (Agent Config Distribution)
**Priority:** Should
**Status:** Validated
**Phase:** Phase 6 (v0.2.0)
**Acceptance:** Admin can update agent config via API; agents receive updated config on next heartbeat.

### R-05: Active Directory Integration
**Priority:** Must
**Status:** Validated
**Phase:** Phase 7 (v0.3.0)
**Acceptance:** ABAC evaluation uses real AD group membership, device trust, and network location from LDAP queries.

### R-06: Fix Integration Tests
**Priority:** Must
**Status:** Validated
**Phase:** Phase 1 (v0.2.0)
**Acceptance:** `cargo test --workspace` passes with no compilation errors.

### R-07: Rate Limiting
**Priority:** Should
**Status:** Validated
**Phase:** Phase 8 (v0.3.0)
**Acceptance:** Brute-force login attempts throttled. Event ingestion has per-agent rate limits. 429 responses include Retry-After header.

### R-08: JWT Secret Configuration
**Priority:** Must
**Status:** Validated
**Phase:** Phase 2 (v0.2.0)
**Acceptance:** Server refuses to start without `JWT_SECRET` set (unless `--dev` flag).

### R-09: Admin Audit Logging
**Priority:** Should
**Status:** Validated
**Phase:** Phase 9 (v0.3.0)
**Acceptance:** Policy CRUD and password changes persisted to `audit_events` table with `EventType::AdminAction`. Queryable via `GET /audit/events?event_type=ADMIN_ACTION`.

### R-10: Connection Pool for SQLite
**Priority:** Could
**Status:** Validated
**Phase:** Phase 10 (v0.3.0)
**Acceptance:** Concurrent API requests execute without serializing on a single mutex. All 220 workspace tests pass.
